# Internationalization (i18n) Guide

## Overview

Diskus uses [i18next](https://www.i18next.com/) with the `react-i18next` integration for multi-language support.

**Supported languages:**
- English (`en`) — default
- Arabic (`ar`) — with RTL layout
- French (`fr`)
- Japanese (`ja`)

---

## Configuration

**File:** `src/i18n.js`

```js
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import Backend from "i18next-http-backend";
import LanguageDetector from "i18next-browser-languagedetector";

i18n
  .use(Backend)           // loads translations from /public/locales/
  .use(initReactI18next)  // React integration
  .use(LanguageDetector)  // auto-detects user language
  .init({
    fallbackLng: "en",    // fallback to English if translation missing
    detection: {
      order: ["path", "localStorage", "htmlTag", "cookie"],
      caches: ["cookie"],
    },
  });
```

### Language Detection Order

i18next detects the user's language in this priority order:

1. **URL path** — e.g., `/ar/Diskus/`
2. **localStorage** — previously saved preference
3. **htmlTag** — `<html lang="...">` attribute
4. **Cookie** — cached language preference

The detected language is cached in a **cookie**.

## Translation Files

Translation files live in `public/locales/<lang>/translation.json`.

```
public/
└── locales/
    ├── en/
    │   └── translation.json   (~167KB, ~5000+ keys)
    ├── ar/
    │   └── translation.json
    ├── fr/
    │   └── translation.json
    └── ja/
        └── translation.json
```

Files are loaded on-demand via `i18next-http-backend` — only the active language file is fetched.

## Using Translations in Components

### `useTranslation` hook

```jsx
import { useTranslation } from 'react-i18next';

const MyComponent = () => {
  const { t } = useTranslation();

  return <h1>{t('dashboard.title')}</h1>;
};
```

### `Trans` component (for JSX with embedded elements)

```jsx
import { Trans } from 'react-i18next';

<Trans i18nKey="welcome.message" values={{ name: userName }}>
  Welcome, <strong>{{ name }}</strong>!
</Trans>
```

## RTL Support

Arabic is a right-to-left language. RTL layout is handled by a dedicated stylesheet:

- **`src/ar.css`** — Arabic-specific CSS overrides (text alignment, flex direction, margin/padding mirroring)
- **`src/fr.css`** — French-specific overrides (if any)

These stylesheets are imported globally in `src/App.js` and apply CSS rules scoped to RTL contexts.

---

## Adding a New Language

1. Create the translation file:
   ```
   public/locales/<lang-code>/translation.json
   ```

2. Populate it with the same keys as `public/locales/en/translation.json`.

3. If the language is RTL, add corresponding CSS rules to a new `src/<lang>.css` file and import it in `src/App.js`.

4. The `LanguageDetector` will automatically pick up the new language — no code changes needed in `i18n.js`.

---

## Adding a New Translation Key

1. Add the key to **all** language files:
   ```json
   // public/locales/en/translation.json
   {
     "myModule": {
       "myKey": "My English text"
     }
   }
   ```

   ```json
   // public/locales/ar/translation.json
   {
     "myModule": {
       "myKey": "النص العربي"
     }
   }
   ```

2. Use it in your component:
   ```jsx
   const { t } = useTranslation();
   t('myModule.myKey');
   ```

## Changing Language Programmatically

The current language is also stored in Redux (`LanguageReducer`). To change language:

```jsx
import i18n from '../i18n';

// Change language
i18n.changeLanguage('ar');
```

Or dispatch the Redux action that handles language change (check `src/store/actions/` for the relevant action).
